===============================================================================
3. TYPOGRAPHY
===============================================================================
Font family:
- Sans / system font stack

Text styles:
- H1: text-3xl, font-extrabold
- H2: text-2xl, font-bold
- H3: text-xl, font-bold
- Body: text-base, font-normal
- Large body: text-lg, font-medium
- Small text: text-sm
- Label: text-xs, font-bold, uppercase, tracking-wider

Line height:
- Headings: tight
- Body: normal to relaxed

===============================================================================
4. SPACING & SIZING
===============================================================================
Tailwind scale is based on a 4px unit.

Common spacing:
- px-3, px-4, px-6
- py-2, py-3, py-4, py-8
- gap-2, gap-3, gap-4, gap-6, gap-12

Common sizing:
- Inputs: padded, full width
- Buttons: compact to medium
- Image cards: h-60
- Icons: w-4 h-4 to w-6 h-6

Containers:
- Mobile padding: px-4 or px-6
- Desktop padding: px-6

===============================================================================
5. BORDER RADIUS
===============================================================================
- rounded-lg: 8px
- rounded-xl: 12px
- rounded-2xl: 16px
- rounded-3xl: 24px
- rounded-full: circular badges and avatars

===============================================================================
6. SHADOWS & DEPTH
===============================================================================
- shadow-sm: subtle cards and borders
- shadow-md: interactive elements
- shadow-lg: hover state
- shadow-xl: elevated cards and modals
- shadow-2xl: popovers and overlays

Common hover behavior:
- cards lift slightly with shadow-xl and -translate-y-1

===============================================================================
7. ANIMATIONS & TRANSITIONS
===============================================================================
Main motion tokens:
- float: vertical floating
- sway: small side-to-side rotation
- sunrays: rotation
- swing: pendulum motion
- wave-drift: horizontal drift
- wave-drift-rev: reverse horizontal drift
- fly-across: crossing motion with fade
- scanline: quick transition effect
- twinkle: opacity pulse

Default transition timing:
- 300ms for most UI transitions
- 700ms for image zoom effects

Common interaction patterns:
- transform and opacity are used for smooth motion
- image hover scales to 110%
- text links change color on hover

===============================================================================
8. COMMON COMPONENT STYLES
===============================================================================
Buttons:
- Primary: bg-[color:var(--app-primary)] text-white rounded-xl px-6 py-3
- Hover: bg-[color:var(--app-primary-hover)]
- Secondary: light surface background with border
- Link button: primary text color, no underline by default

Cards:
- Background: white surface
- Border: app border color
- Radius: rounded-2xl or rounded-3xl
- Shadow: shadow-sm by default, shadow-lg on hover

Inputs:
- Background: white surface
- Border: app-border-strong
- Radius: rounded-xl
- Padding: px-3 py-3 or px-4 py-3
- Focus: ring-2 with app-primary-soft and app-primary border

Badges:
- Rounded-full
- Text-xs, font-bold, uppercase
- Used for room status and small labels

Room card status colors:
- AVAILABLE: bg-emerald-50, text-emerald-600
- RENTED: bg-rose-50, text-rose-600
- MAINTENANCE: bg-amber-50, text-amber-600

Header:
- White background
- Bottom border using app-border
- Horizontal layout with navigation and user actions

Footer:
- White background
- Top border using app-border
- Multi-column layout on desktop
- Muted text for links and copyright

===============================================================================
11. RESPONSIVE RULES
===============================================================================
- Mobile-first layout
- Use sm, md, lg prefixes for larger screens
- Keep tap targets at least 44x44px
- Increase spacing and font size on desktop
- Hide secondary navigation on small screens when needed

===============================================================================
12. ACCESSIBILITY
===============================================================================
- Maintain strong text contrast
- Keep focus rings visible
- Use semantic headings and form labels
- Use aria labels for icon-only controls
- Use descriptive alt text for room images

===============================================================================
13. FORM STATES
===============================================================================
Error:
- border-red-500 or border-rose-600
- bg-red-50
- text-red-700

Success:
- border-emerald-500
- bg-emerald-50
- text-emerald-700

Warning:
- border-amber-500
- bg-amber-50
- text-amber-700

Info:
- border-blue-500
- bg-blue-50
- text-blue-700

Loading:
- spinner or skeleton placeholder
- disabled button state

===============================================================================
15. PERFORMANCE NOTES
===============================================================================
- Use CSS variables for design tokens
- Prefer transform and opacity for animations
- Avoid heavy effects on hover
- Keep image sizes optimized
- Use Tailwind utilities instead of custom CSS where possible

===============================================================================

- Create component variants for buttons, inputs, cards, and badges
